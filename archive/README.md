# Archive — superseded artifacts

## Corpus unification (2026-07-17)

Two real-AAAI-27-policy chunkings were built in parallel on different branches
and met at the 2026-07-17 merge. **Decision: the Phase-7C chunking is
canonical** (93 chunks, ids `policy_101`–`policy_193`, subsection-level splits
with contextual path titles, generated by `backend/scripts/chunk_policies.py`
from `data/policy_corpus_real/`). It now lives at
`data/knowledge_base/policies.json` and is what the app seeds and retrieves
against. It was chosen because the Phase-7D real-ticket retrieval benchmark
(`docs/exp_tracking/E001_retrieval_testing.md`) and the running demo app were
built on it.

Archived here from the other track (Real-Corpus phase, merged via PR #1):

| file | what it was |
|---|---|
| `policies_real_56chunks.json` | The competing chunking — 56 chunks, ids `policy_046`–`policy_101`, section-level splits, 25-term tag taxonomy. Ids collide with the canonical corpus at `policy_101`. |
| `policies_real_staged.json` | Staging copy of the same corpus used during its swap-in. |
| `ground_truth_remap_proposal.json` | The proposal used to remap the synthetic eval set onto the 56-chunk ids. |

Kept from that track (complementary, still in use): the `tags`/`source` DB
columns (migration `b8d3f6a1c204`), tag parity in the FAISS/BM25 retrievers and
policy repository, and the in-repo policy source docs
(`data/policy_corpus_real/`).

**Known residue:** `data/eval/ground_truth.json`'s `relevant_chunk_id` values
still reference the archived `policy_046`–`101` ids. Its classification and
routing labels remain valid (corpus-independent); its retrieval-only metrics
are stale and the synthetic set was retired from retrieval benchmarking in
Phase 7D anyway (real-ticket labels in `data/eval_real/` are the benchmark
now). Remap against the canonical ids only if synthetic retrieval metrics are
ever needed again.
